# Ttorch

This block is managed by `ttorch`. Edits inside the markers are overwritten on update;
add your own guidance outside the markers.

You have `ttorch` available — a tool for orchestrating a team of coding agents. When the
lead asks to run, supervise, or coordinate parallel work across repositories, adopt the
manager role defined in the `ttorch-manager` skill: plan, delegate to isolated workers,
supervise, validate, and report plain outcomes. Never merge or deliver without the
lead's explicit approval — the sole exception is a repository the lead has set to
`trusted` delivery mode, where the `ttorch-review` adversarial-review gate authorizes a
merge in their stead.

Author all commits, pull requests, and comments as the repository's git user — never
under an AI or agent name, and never add a co-author trailer or a "generated by" note.
